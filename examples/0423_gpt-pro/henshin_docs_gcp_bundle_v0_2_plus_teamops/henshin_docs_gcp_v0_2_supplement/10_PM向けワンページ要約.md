# 10_PM向けワンページ要約

Version: 0.2.1  
用途: 素人PMが、いま何が大事で、何を見ればいいかを1ページで把握する

---

## 1. このプロジェクトは何か
- AIで感情や文脈を読み取る
- Webでスーツを組み立てる
- Questで変身試験をする
- Quest内でReplayを見る

つまり、**Webでスーツ設計、Questで試験と再生**の2段構え。

---

## 2. いま最優先で決めること
1. SuitManifest の中身
2. PartCatalog の中身
3. TransformEvent の種類
4. Firestore と Cloud SQL の境界

これが決まると、Web / Backend / Quest が並行で動ける。

---

## 3. 逆に、いま決めなくていいこと
- UIの細かい見た目
- WebXR本編
- 完全自動3D生成
- 動画書き出しの豪華さ
- MCPの深い導入

---

## 4. いまの壁
- スーツ正本の schema がまだ粗い
- パーツ辞書がまだない
- Quest Replay の仕様が粗い
- DBの役割分担がまだ言葉レベル

---

## 5. いまの次の一手

- エンジニアが backend skeleton を立てる
